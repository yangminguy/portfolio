# CONTENT GUIDE — Portfolio Deck Copy

## Writing Principle

This website should be concise and editorial.

Do not over-explain on the homepage.
Use project detail pages for depth.

## Homepage Copy Rules

Good homepage copy should be:

- short
- specific
- confident
- calm
- evidence-oriented

Avoid:

- long self-introductions
- generic passion statements
- unsupported metrics
- buzzwords without context
- dense paragraphs inside slides

## Recommended Main Positioning

```text
I build systems for scattered work.
```

Korean support line example:

```text
흩어진 실행, 문서, 사람, 데이터를 반복 가능한 구조로 정리합니다.
```

## Project Index Format

Each project should have:

```text
Title — Year
One-line summary
Role / domain
```

Example:

```text
Campit Influencer Marketing System — 2024
Turning influencer collaborations into a repeatable marketing operation system.
Marketing Operations / No-code Automation / Creator Ops
```

## Detail Page Structure

Use this structure for each project:

1. Context
2. Problem
3. System
4. Execution
5. Impact
6. Artifacts
7. Reflection

## Claim Safety

Do not write:

```text
I single-handedly increased revenue by 30%.
```

Prefer:

```text
Participating sites showed meaningful reservation and sales lift during campaign periods, based on internal tracking.
```

Use exact numbers only when the user confirms them.
